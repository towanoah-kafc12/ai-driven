# decision 記録テンプレート

エージェントとユーザの対話で決定したこと、新たに判明した重要情報、判断理由、設計上の前提を記録します。

## ファイル名

新しく記録ファイルを作る場合は、機能名・作業単位・調査対象が分かる kebab-case または snake_case の名前に、用途 suffix `_decision.md` を付ける。

例: `auth_decision.md`, `wsl-context7-mcp_decision.md`

## 必須項目

- 項番
- 日付
- タイトル
- 内容

## 記載形式

1. `## <日付>: <タイトル>` のセクション単位で記載する。
2. セクション内に項番、内容、判断理由を含める。
3. 対応が必要なファイル、関連情報、代替案、影響範囲がある場合は、必要に応じて小セクションを追加する。

## ひな形

```markdown
## YYYY-MM-DD: タイトル

- 項番: 1
- 内容: 
- 判断理由: 

### 対応が必要なファイル

- 

### 関連情報

- 

### 代替案

- 

### 影響範囲

- 
```
