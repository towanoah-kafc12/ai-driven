# review 記録テンプレート

エージェントのレビュー結果、ユーザからの追加指摘、未対応の改善指摘を記録します。

## ファイル名

新しく記録ファイルを作る場合は、機能名・作業単位・調査対象が分かる kebab-case または snake_case の名前に、用途 suffix `_review.md` を付ける。

例: `auth_review.md`, `template-operations_review.md`

## 必須項目

- 項番
- 日付
- 対象ファイル
- 指摘内容
- 対応方針
- 状態（- / 修正済）

## 記載形式

1. ファイル上部に Markdown の表を置き、項番、日付、対象ファイル、指摘内容、対応方針、状態を一覧化する。
2. 指摘内容が複雑または長くなる場合は、ファイル下部に `## <項番>: <タイトル>` のセクションを作り、詳細を記載する。
3. 修正が完了したら状態を `修正済` に更新し、必要に応じて関連する decision も更新する。

## ひな形

```markdown
| 項番 | 日付 | 対象ファイル | 指摘内容 | 対応方針 | 状態 |
| --- | --- | --- | --- | --- | --- |
| 1 | YYYY-MM-DD |  |  |  | - |

## 1: タイトル

### 指摘内容


### 対応方針


### 対応結果


```
