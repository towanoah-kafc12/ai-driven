# ai-driven

AI 駆動開発を、場当たり的なチャットではなく、再利用できるドキュメントと意思決定の流れとして運用するためのテンプレート集です。

このリポジトリでは、アイデアの詳細化、実装方針の整理、タスク分解、検証、レビューを段階的に記録し、人間と AI エージェントが同じ前提を参照しながら作業できる状態を目指します。

## 目的

- アイデア段階の曖昧な要望を、実装可能な単位まで整理する
- 未確定事項、決定事項、レビュー指摘を混ぜずに管理する
- AI エージェントに作業を依頼するときの入力、出力、完了条件を明確にする
- 実装前後の判断や検証をドキュメントとして残す

### `AGENTS.md`

AI エージェントが作業するときの入口です。作業開始時の確認事項、ドキュメントの読み方、記録先、レビューや検証の扱いを定義します。

### `docs/00_base/`

AI 駆動開発で使う基本テンプレートです。

- `00_idea.md`: アイデア詳細化テンプレート
- `01_implementation.md`: 実装方針テンプレート
- `02_task_breakdown.md`: タスク分解テンプレート
- `03_verification.md`: 検証テンプレート

### `docs/01_pending/`

未確定事項、確認待ち、保留中の論点を記録します。

チャット上で決めきれない内容や、後で判断が必要な内容はここに残します。

### `docs/02_decision/`

決定済みの内容を記録します。

なぜその方針にしたのか、どのファイルに影響するのか、代替案は何だったのかを残すことで、後続作業の前提を明確にします。

### `docs/03_review/`

レビュー指摘、対応方針、対応結果を記録します。

実装やドキュメント更新のあとに、未対応の指摘や改善点がチャットだけに残らないようにするための領域です。

## 基本ワークフロー

```text
idea
  ↓
implementation policy
  ↓
task breakdown
  ↓
verification
  ↓
review
```

1. **アイデアを整理する**
   - `docs/00_base/00_idea.md` を使って、目的、想定ユーザ、MVP、非目標、成功条件を整理します。

2. **実装方針を決める**
   - `docs/00_base/01_implementation.md` を使って、技術選定、アーキテクチャ、データ設計、UI / UX、テスト方針をまとめます。

3. **タスクに分解する**
   - `docs/00_base/02_task_breakdown.md` を使って、作業単位、変更予定ファイル、完了条件、確認方法を明確にします。

4. **検証する**
   - `docs/00_base/03_verification.md` を使って、受け入れ条件、自動テスト、手動確認、既存機能への影響を確認します。

5. **レビューする**
   - `docs/03_review/` にレビュー結果を記録し、指摘内容、対応方針、対応結果を追跡します。

## 記録の使い分け

| 状態 | 記録先 | 例 |
| --- | --- | --- |
| まだ決めていない | `docs/01_pending/` | 技術選定の候補、仕様の確認待ち、未解決の懸念 |
| 決めた | `docs/02_decision/` | 採用技術、実装方針、スコープ、非目標 |
| 見直した | `docs/03_review/` | レビュー指摘、修正方針、対応結果 |

## AI エージェントとの協働

AI エージェントに作業を依頼するときは、次の情報をできるだけ明確にします。

- 参照すべき `docs/` 配下のファイル
- 今回のゴール
- 変更してよい範囲
- 完了条件
- 検証方法
- 未確定事項や既存の決定事項

詳細な運用ルールは `AGENTS.md` を参照してください。

## 現在のステータス

このリポジトリは、AI 駆動開発のためのドキュメントテンプレートと運用ルールを整備している段階です。

現時点ではアプリケーションのビルド手順や実行コマンドではなく、ドキュメントを中心に構成されています。
