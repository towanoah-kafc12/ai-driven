# AI駆動開発テンプレート運用ガイド

このリポジトリは、アイデアをAIエージェントに渡してから、要件整理・意思決定・実装・レビューまでを毎回同じ品質で進めるためのテンプレートです。作業するエージェントは、本ファイル、`MEMO.md`、および `docs/` 配下の関連記録を確認し、判断根拠や未解決事項を必要な場所に残してください。

## 基本方針

- 作業開始時に、本ファイル、`MEMO.md`、および `docs/` 配下の関連ファイルを確認する。
- 不明点、意思決定、レビュー指摘は口頭やチャットだけで終わらせず、該当する `docs/` 配下の記録に反映する。
- 実装・修正時は、関連する pending / decision / review の記録を参照し、未解決事項や既存の決定に反しないことを確認する。
- `AGENTS.md` を更新した場合は、原則として同じ内容を `CLAUDE.md` にも反映する。

## docs ディレクトリの役割

`docs/` 配下には、AI駆動開発中に発生する未解決事項、決定事項、レビュー指摘を蓄積します。詳細な書き方は、各フォルダの `template.md` を参照してください。

- `docs/00_base/00_idea.md`: アイデアだけの状態から、目的、想定ユーザ、MVP、非目標、成功条件を整理する。
- `docs/00_base/01_implementation.md`: 技術、設計方針、実装方針、実装スコープ、テスト方針を整理する。
- `docs/00_base/02_task_breakdown.md`: 実装方針を、AIエージェントが安全に実行できる作業単位へ分解する。
- `docs/00_base/03_verification.md`: 実装後に、目的、MVP、タスク完了条件、テスト方針を満たしているか確認する。
- `docs/01_pending/`: 不明点、情報不足、ユーザ確認が必要な事項、調査待ちの事項を記録する。書き方は `docs/01_pending/template.md` を参照する。
- `docs/02_decision/`: エージェントとユーザの対話で決定したこと、新たに判明した重要情報、判断理由、設計上の前提を記録する。書き方は `docs/02_decision/template.md` を参照する。
- `docs/03_review/`: エージェントのレビュー結果、ユーザからの追加指摘、未対応の改善指摘を記録する。書き方は `docs/03_review/template.md` を参照する。

## pending / decision / review の記録トリガー

### pending に記録する時

- 実装や判断に必要な情報が不足している。
- ユーザ確認が必要で、現時点では決めきれない。
- 外部仕様、既存挙動、制約などの調査待ちがある。
- 未解決のまま作業を進めると、後続の判断や実装がぶれる可能性がある。

### decision に記録する時

- ユーザとの対話で方針、スコープ、制約、非目標が決まった。
- 技術選定、設計方針、運用ルールなど、後続作業の前提になる判断をした。
- 調査によって重要な事実や前提が判明した。
- 代替案を比較し、採用しない理由も残しておく必要がある。

### review に記録する時

- エージェントレビュー、ユーザ指摘、手動確認で改善点や懸念が出た。
- すぐに直さないが、後で対応すべき指摘がある。
- 修正済みかどうかを追跡したいレビュー項目がある。
- decision に影響する指摘や、今後のテンプレート改善につながる指摘がある。

## 作業時の確認事項

- 関連する `docs/00_base/` の前提、実装方針、タスク分解、検証方針を確認する。
- 関連する `docs/01_pending/` の未解決事項を確認する。
- 関連する `docs/02_decision/` の決定事項を確認する。
- 関連する `docs/03_review/` の未対応指摘を確認する。
- 新しい不明点、決定、レビュー指摘が発生した場合、該当するフォルダの `template.md` に従って記録する。
